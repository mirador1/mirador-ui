# `src/app/features/` — Routed pages

Each subdirectory is a **lazy-loaded** feature, wired in `src/app/app.routes.ts`
via `loadComponent: () => import('./features/X').then(m => m.XComponent)`.
This means each feature's JS is fetched only when the user navigates to it.

## Conventions

- One folder per feature. Folder name = route path = feature name.
- Each folder ships a `<name>.component.ts`, `.html`, `.scss`, `.spec.ts`.
  Larger features may have helpers/subcomponents in the same folder.
- Features must not import from each other. Shared logic goes in `../core/`
  or `../shared/`.
- Long-form prose (help text longer than ~3 sentences) should live in
  `../../docs/` and be referenced from the component, not inlined. See the
  ongoing doc-extraction effort in TASKS.md.
