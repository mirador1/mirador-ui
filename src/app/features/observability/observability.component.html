ℹ️ What is SSE? — Server-Sent Events is a browser API where the server keeps an HTTP connection open and pushes lines of text to the client in real-time, without the client polling. The client opens one connection with new EventSource('/stream') and receives data whenever the server sends it. Example: the Dashboard's "Live Feed" section uses SSE to receive new customer creation events the moment they happen.

⚠️ This tab is different — it uses polling, not SSE. Every 2 seconds the UI calls GET /actuator/prometheus (a plain HTTP endpoint), parses the raw Prometheus text format to extract HTTP request counters per endpoint/method/status, then computes the delta since the previous poll and shows only endpoints with new traffic. It is not a push stream — the backend is not aware of the UI watching.

Color coding: 🟢 2xx success   🟠 4xx client error   🔴 5xx server error
